# 祇園町火災検知AI

京都・祇園町南側の景観を想定した、火災・煙検知AIアプリです。  
画像をアップロードすると、YOLOv8による物体検出とMobileNetV2系画像分類モデルを組み合わせて、火災リスクを判定します。

祇園町では、赤い提灯・暖色照明・料理煙・湯気などが多く、一般的な火災検知AIでは誤検知が起きやすいと考えました。  
そのため、本アプリでは単純に「赤い＝火」と判定するのではなく、複数モデルの結果を統合し、赤色照明反応や料理煙の可能性を考慮する設計にしています。

---

## 1. 制作目的

このアプリは、就職活動用ポートフォリオとして制作したDjangoアプリです。

目的は以下の3つです。

1. DjangoでAI画像判定アプリを実装すること
2. YOLOv8と画像分類モデルを組み合わせた判定ロジックを作ること
3. 祇園町という地域特性に合わせた誤検知対策を設計すること

単にAIモデルを動かすだけではなく、実際の運用で問題になりやすい「誤検知」をどう減らすかを重視しています。

---

## 2. 主な機能

### 画像アップロード判定

ブラウザから画像をアップロードし、AIで火災・煙の可能性を判定します。

対応している判定モードは以下です。

| モード | 内容 |
|---|---|
| 統合判定モード | Gion YOLO、公開Fire-Smoke YOLO、画像分類AIを組み合わせて判定 |
| 物体検出モード | 自作Gion YOLOで fire / smoke の位置を検出 |
| 画像分類モード | MobileNetV2系モデルで画像全体を fire / normal / smoke に分類 |

---

### 統合判定

本アプリでは、以下の3つのAI判定結果を統合します。

1. 自作Gion YOLOモデル
2. 公開Fire-Smoke YOLOモデル
3. MobileNetV2系画像分類モデル

特に、祇園町で起こりやすい以下の誤検知を考慮しています。

| 誤検知要因 | 対策 |
|---|---|
| 赤い提灯 | YOLOが火や煙を検出していない場合は赤色照明反応として扱う |
| 暖色照明 | 画像分類AIだけのfire判定を火災確定にしない |
| 料理煙・湯気 | smoke判定や面積情報を見て、火災とは別の要確認判定にする |
| 夜間の赤い反射 | 位置検出がない場合はリスクを下げる |

---

### 検出枠付き画像の表示

YOLOが fire / smoke を検出した場合、検出枠付き画像を画面に表示します。

表示内容は以下です。

- 検出ラベル
- 信頼度
- 検出面積
- 座標
- 検出枠付き画像

---

### フィードバック保存機能

AI判定後、ユーザーが判定結果に対してフィードバックを保存できます。

保存できるカテゴリは以下です。

| カテゴリ | 意味 |
|---|---|
| この判定は正しい | AI判定が正しかった画像 |
| 誤検知：赤い提灯・赤色照明 | 赤い提灯や照明を火と誤判定した画像 |
| 誤検知：料理煙・湯気 | 火災ではない煙や湯気を誤検知した画像 |
| 見逃し：火・煙がある | 火や煙があるのにAIが弱く判定した画像 |
| その他 | 上記以外の改善用データ |

保存された画像とJSONは、以下に保存されます。

```txt

├── correct/
├── false_positive_red_lantern/
├── false_positive_non_fire_smoke/

└── other/

フィードバック履歴画面

保存済みフィードバックを一覧画面で確認できます。

URLは以下です。

一覧画面では、以下を確認できます。

これにより、次回の再学習データ作成に使える画像を整理できます。

4. 使用モデル
画像分類モデル
prediction/models/260627model_gion_fire_MNV2.h5.keras

画像全体を以下の3クラスに分類します。

fire
normal
smoke
自作Gion YOLOモデル
prediction/models/gion_fire_yolov8n_finetune_v2_best.pt

祇園町向けデータセットで学習したYOLOv8モデルです。
火や煙の位置検出に使用します。

クラスは以下です。

0 = smoke
1 = fire
公開Fire-Smoke YOLOモデル
prediction/models/public_fire_smoke_yolov8n_best.pt

補助的な火・煙検出モデルとして使用します。

このモデルは、自作Gion YOLOだけでは拾いにくい煙検出の補助として利用しています。
ただし、公開モデルは赤色や発光部分に反応する可能性があるため、火災確定判定には強く使いすぎない設計にしています。


使い方
画像判定
トップ画面を開く
画像をドラッグ＆ドロップ、またはクリックして選択
判定モードを選ぶ
「AIで判定する」を押す
判定結果、判定理由、検出枠付き画像を確認する

判定ロジックの考え方

このアプリでは、1つのAIモデルだけで判定を決めません。

なぜ統合判定にしたか

祇園町では、火災ではないのに火に見えやすいものが多くあります。

例：

赤い提灯
赤い看板
暖色照明
料理煙
湯気
夜間の反射

画像分類AIは画像全体の色や雰囲気を見て判断するため、赤い提灯を火と判断する場合があります。
そこで、YOLOの位置検出結果と組み合わせて、火や煙の場所が検出されているかを確認します。

赤色照明反応の判定

以下の条件では、火災リスクを強く扱いすぎないようにしています。

画像分類AI = fire
Gion YOLO = 検出なし
公開Fire-Smoke YOLO = 検出なし

この場合、最終判定は以下のようにします。

赤い提灯・赤色照明の可能性があります

これにより、祇園町特有の赤い照明による誤検知を抑えます。

9. 工夫した点
地域特性を考慮したこと

一般的な火災検知ではなく、祇園町南側という地域に合わせて設計しました。

特に以下を意識しています。

木造町家が多い
赤い提灯が多い
夜間照明が多い
飲食店の料理煙が出やすい
観光地として人通りがある
誤検知を保存できるようにしたこと

AIは最初から完璧ではないため、誤検知画像を保存し、次回の改善につなげられるようにしました。

この機能により、以下の改善サイクルを作れます。

AI判定
↓
誤検知・見逃しを保存
↓
フィードバック履歴で確認
↓
再学習データとして整理
↓
モデル改善
判定理由を表示したこと

単に「火災です」と表示するのではなく、なぜその判定になったかを表示します。

表示例：

Gion YOLOのfire最大信頼度
Gion YOLOのsmoke最大信頼度
公開Fire-Smoke YOLOの判定
画像分類AIの判定
YOLO未検出・分類AIのみfire判定
煙優勢判定

これにより、AI判定の透明性を高めています。

10. 現在の課題

現時点では、以下の課題があります。

課題	内容
モデル精度	赤い提灯や暖色照明への誤判定は完全には解消していない
データ数	祇園町特化の実画像データはまだ十分ではない
保存方式	フィードバック履歴はDBではなくファイル保存
UI	今後、Liquid Glass風UIやマップ表示に改善予定
実運用	現在は画像アップロード判定であり、リアルタイム監視ではない
